?先看这里：这份稿怎么用、"切图/像素"是什么

这份页面就是给研发的交付物。下面每一屏都是真实网页（不是截图），前端在浏览器里右键「检查」就能直接读到每个元素的颜色、字号、间距、圆角——等于「标注稿」和「设计规范」二合一，不用再单独标注。

要不要写进 Figma？不建议、我也不该动你们设计师的 Figma 源文件。正规做法是：这些稿定了之后，交给设计师在 Figma 里重建成设计源，之后用 Figma 的 Dev Mode 自动出标注和切图。但那需要设计师排期；而这些稿本身已经是精确的 HTML/CSS，前端拿这份交付页可以立刻开工，不必等 Figma。

"切图"是什么：把设计里代码画不出来的图片素材（图标、插画、logo）导出成图片文件给前端。好消息：这套稿几乎不用切图——渐变、圆角、色块、进度环全是 CSS 画的，前端直接写代码即可；只有图标需要单独给（见最下方清单），给 SVG 就行。
"像素/倍图"是什么：手机设计按"逻辑尺寸"算（iPhone 逻辑宽 375pt），但真机屏幕像素更密（2 倍、3 倍）。所以图片素材要导出 @2×(750px)、@3×(1125px) 两三套才不糊；代码画的东西不受影响，CSS 自动适配。
⚠️ 本交付页/PNG 都是按 750px（=375pt 的 2 倍） 出的，所以下面规范里的逻辑 pt = 像素 ÷ 2。前端按 1pt=1dp 写布局即可。
